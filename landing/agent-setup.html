Quick setup

Up and running in 5 steps

01

Sign up

Choose a plan and complete checkout. No trial period friction — you get your API key immediately.

Starter — $19/mo    Pro — $49/mo

02

Get your API key

After checkout you land on /welcome. Your API key is displayed there. Copy it — you'll need it for every API call.

Confirm this works

Run the command below. If you see your account info, the key is valid.

terminal 
curl https://api.mentionwatch.com/api/account \
  -H "Authorization: Bearer YOUR_API_KEY"
If you get 401

The key is wrong or you're still on the /welcome page. Copy it again — no spaces, no quotes.

03

Add your first keyword

Replace your-product-name with whatever you want to track. Exact match, case-insensitive.

terminal 
curl -X POST https://api.mentionwatch.com/api/keywords \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"keyword": "your-product-name"}'
Expected response

{"id": "kw_...", "keyword": "your-product-name", "created_at": "..."}

If you get 409

That keyword already exists on your account. Safe to move on.

04

Set your webhook URL

This is where MentionWatch will POST when a mention is found. Must be a publicly reachable HTTPS endpoint.

No server yet? Use webhook.site to get a temporary URL for testing.

terminal 
curl -X POST https://api.mentionwatch.com/api/webhooks \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"url": "https://your-app.com/webhook"}'
05

Test the connection

Sends a synthetic mention payload to your webhook URL right now — no need to wait for a real mention.

terminal 
curl -X POST https://api.mentionwatch.com/api/webhooks/test \
  -H "Authorization: Bearer YOUR_API_KEY"
You're done

Check your webhook endpoint for the test payload. Real mentions arrive within 15 minutes of being posted.

Security

Webhook verification

Every webhook request includes an X-MentionWatch-Signature header. It's an HMAC-SHA256 hex digest of the raw request body, signed with your API key. Verify it before processing.

webhook.js — Node.js 
// Express example
const crypto = require('crypto');

function verifySignature(rawBody, signature, apiKey) {
  const expected = crypto
    .createHmac('sha256', apiKey)
    .update(rawBody)
    .digest('hex');

  // constant-time comparison prevents timing attacks
  return crypto.timingSafeEqual(
    Buffer.from(expected),
    Buffer.from(signature)
  );
}

app.post('/webhook', express.raw({ type: 'application/json' }), (req, res) => {
  const sig = req.headers['x-mentionwatch-signature'];
  if (!verifySignature(req.body, sig, process.env.MENTIONWATCH_API_KEY)) {
    return res.status(401).send('Invalid signature');
  }
  const mention = JSON.parse(req.body);
  // handle mention.keyword, mention.url, mention.text, mention.source
  res.sendStatus(200);
});
webhook.py — Python / Flask 
import hmac, hashlib, os
from flask import Flask, request, abort

app = Flask(__name__)

def verify_signature(raw_body: bytes, signature: str) -> bool:
    api_key = os.environ["MENTIONWATCH_API_KEY"].encode()
    expected = hmac.new(api_key, raw_body, hashlib.sha256).hexdigest()
    return hmac.compare_digest(expected, signature)

@app.route("/webhook", methods=["POST"])
def webhook():
    sig = request.headers.get("X-MentionWatch-Signature", "")
    if not verify_signature(request.get_data(), sig):
        abort(401)
    mention = request.get_json(force=True)
    # handle mention["keyword"], mention["url"], mention["text"], mention["source"]
    return "", 200
