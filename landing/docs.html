Authentication

All requests require a Bearer token in the Authorization header. Keys are prefixed mw_live_ for production and mw_test_ for sandbox.

curl https://api.mentionwatch.com/api/me \
  -H "Authorization: Bearer mw_live_your_api_key"

Your API key is shown on your welcome page after signup. Treat it like a password — it grants full access to your account.

Quickstart

Three commands to start receiving webhook notifications. Takes about 90 seconds.

1. Create a keyword
POST /api/keywords 
curl -X POST https://api.mentionwatch.com/api/keywords \
  -H "Authorization: Bearer mw_live_xxx" \
  -H "Content-Type: application/json" \
  -d '{"keyword": "my-product"}'

# 201 Created
{
  "data": {
    "id": "kw_abc123",
    "keyword": "my-product",
    "created_at": "2026-04-11T10:00:00Z"
  },
  "meta": { "remaining_keywords": 4 }
}
2. Set your webhook URL
POST /api/webhooks 
curl -X POST https://api.mentionwatch.com/api/webhooks \
  -H "Authorization: Bearer mw_live_xxx" \
  -H "Content-Type: application/json" \
  -d '{"url": "https://your-app.com/mentions-webhook"}'

# 200 OK
{
  "data": {
    "url": "https://your-app.com/mentions-webhook",
    "updated_at": "2026-04-11T10:00:05Z"
  }
}
3. Send a test payload
POST /api/webhooks/test 
curl -X POST https://api.mentionwatch.com/api/webhooks/test \
  -H "Authorization: Bearer mw_live_xxx"

# 200 OK
{ "data": { "status": "sent", "url": "https://your-app.com/mentions-webhook" } }

# Your endpoint receives a POST with a sample payload immediately.

Webhook Payload

Every mention triggers an HTTP POST to your webhook URL with this JSON body:

{
  "id":           "mw_evt_k8x92mf",
  "keyword":      "my-product",
  "source":       "hackernews",
  "type":         "story",
  "title":        "Show HN: My Product does X better",
  "text":         "We built this because existing tools...",
  "author":       "hackernewsuser",
  "url":          "https://news.ycombinator.com/item?id=42345678",
  "source_url":   "https://example.com/my-product",
  "timestamp":    "2026-04-11T10:30:00Z",
  "delivered_at": "2026-04-11T10:35:12Z"
}
Headers
Header Value
Content-Type application/json
X-MentionWatch-Signature sha256=<hmac>
X-MentionWatch-Event-Id mw_evt_k8x92mf
Fields
Field Type Description
id string Unique event ID
keyword string The keyword that was matched
source string "hackernews" or "reddit"
type string "story", "post", "comment"
title string Post or story title
text string Body text (max 4096 chars)
author string Username of the author
url string Direct link to the mention
source_url string? Linked article URL, if any
timestamp string ISO 8601 when the mention was posted
delivered_at string ISO 8601 when the webhook was sent

Webhook Verification

Every webhook includes an HMAC-SHA256 signature in the X-MentionWatch-Signature header. Verify it to confirm the payload came from MentionWatch and has not been tampered with.

Node.js
webhook.js 
import crypto from 'crypto';

function verifyWebhook(body, signature, secret) {
  const expected = crypto
    .createHmac('sha256', secret)
    .update(body)
    .digest('hex');
  return signature === `sha256=${expected}`;
}

// In your webhook handler:
app.post('/webhook', (req, res) => {
  const sig = req.headers['x-mentionwatch-signature'];
  const raw = JSON.stringify(req.body);

  if (!verifyWebhook(raw, sig, process.env.MW_WEBHOOK_SECRET)) {
    return res.status(401).json({ error: 'Invalid signature' });
  }

  // Process the mention
  console.log('Mention:', req.body.keyword, req.body.source);
  res.status(200).json({ received: true });
});
Python
webhook.py 
import hmac, hashlib

def verify_webhook(body: bytes, signature: str, secret: str) -> bool:
    expected = hmac.new(
        secret.encode(), body, hashlib.sha256
    ).hexdigest()
    return hmac.compare_digest(signature, f"sha256={expected}")

# In your Flask handler:
@app.route('/webhook', methods=['POST'])
def webhook():
    sig = request.headers.get('X-MentionWatch-Signature', '')
    if not verify_webhook(request.data, sig, WEBHOOK_SECRET):
        return {'error': 'Invalid signature'}, 401

    data = request.json
    print(f"Mention: {data['keyword']} on {data['source']}")
    return {'received': True}

Error Codes

All errors return a consistent envelope:

{
  "error": {
    "code":        "keyword_limit_exceeded",
    "message":     "Starter plan allows 5 keywords. You have 5.",
    "upgrade_url": "/pricing",
    "docs":        "/docs#limits"
  }
}
Code HTTP Description
auth_required 401 Missing Authorization header
invalid_api_key 401 API key not found or revoked
keyword_limit_exceeded 403 Plan keyword limit reached
mention_quota_exceeded 429 Monthly mention quota used
rate_limit 429 Too many API requests (100/min)
invalid_keyword 400 Keyword validation failed
duplicate_keyword 409 Keyword already exists
keyword_not_found 404 Keyword ID not found
invalid_webhook_url 400 URL must be public HTTPS
webhook_not_configured 400 Set webhook URL before testing
internal_error 500 Server error (we are notified automatically)

Webhook retry policy

If your endpoint returns a non-2xx response or times out (10s), we retry up to 3 times with exponential backoff: 10 seconds, 60 seconds, then 5 minutes. After 3 failures, the mention is stored and available via GET /api/mentions.
